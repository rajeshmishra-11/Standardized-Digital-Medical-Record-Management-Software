âš™ï¸ Functionality Overview

1. Roles & Authentication

Three roles: patient, doctor, pharmacy

JWT-based login and protected routes

AuthContext.js manages token and role globally

2. Patient Dashboard (No Search Box)

Displays own data fetched via /api/me/*

Tabs:

My Prescriptions

My Reports

My Profile

My Consents

My Downloads (PDF/JSON export)

3. Doctor Dashboard

Has SearchBar for patient lookup by patientId

Fetches:

/api/patients/:patientId (demographics)

/api/prescriptions?patientId=...

Can issue new prescriptions

4. Pharmacy Dashboard

Has SearchBar for patientId

Fetches prescriptions to verify & mark as dispensed

5. Consent System

Patient grants temporary access via ShareConsentModal

Backend enforces access using consentGuard.js

Method	Endpoint	Role	Description
POST	/api/auth/login	All	JWT login
GET	/api/me/profile	Patient	View own profile
GET	/api/me/prescriptions	Patient	View own prescriptions
GET	/api/me/reports	Patient	View own reports
GET	/api/patients/:patientId	Doctor/Pharmacy	Fetch patient demographics
GET	/api/prescriptions?patientId=...	Doctor/Pharmacy	Fetch prescriptions by patient
POST	/api/prescriptions	Doctor	Create new prescription
PATCH	/api/prescriptions/:rxId/dispense	Pharmacy	Mark prescription as dispensed
POST	/api/consents	Patient	Create consent for doctor/pharmacy
PATCH	/api/consents/:id/revoke	Patient	Revoke consent
